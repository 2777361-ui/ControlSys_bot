# Деплой на Amvera (одно приложение: веб + бот, SQLite)

Одно приложение в Amvera: в одном контейнере работают **веб-интерфейс** (FastAPI на порту 80) и **Telegram-бот**. База данных — **SQLite** в папке `/data` (постоянное хранилище Amvera): общая для веба и бота, не затирается при пересборке. PostgreSQL не нужен.

## Кратко после деплоя

1. **Переменные окружения:**  
   `BOT_TOKEN`, `WEB_SECRET_KEY`, `BASE_URL`.  
   По желанию для первого входа: `INIT_ADMIN_EMAIL`, `INIT_ADMIN_PASSWORD`, `INIT_ADMIN_FULL_NAME` — при старте один раз создаётся администратор (если его ещё нет).

2. **Первый администратор**  
   Задай `INIT_ADMIN_*` и перезапусти приложение — при старте один раз создастся администратор (если его ещё нет).

3. **Вход**  
   Открой URL приложения в браузере — веб-интерфейс. Войди по email и паролю администратора. Бот в Telegram работает параллельно, общая с вебом SQLite-база в `/data/school.db`.

---

## Шаг 1: Репозиторий

Код в Git, привязан к Amvera. В корне: `amvera.yml`, `start.sh`, `requirements.txt`. После пуша в основную ветку (например `master`) Amvera соберёт и запустит приложение.

---

## Шаг 2: Одно приложение в Amvera

1. В личном кабинете Amvera создай **одно** приложение (тип «Приложение»).
2. Подключи репозиторий с проектом.
3. Конфигурация берётся из `amvera.yml`: Python (pip), сборка по `requirements.txt`, запуск через `start.sh`, постоянное хранилище монтируется в `/data`. Менять ничего не нужно.
4. В разделе **Переменные окружения** задай:

| Переменная            | Обязательно | Описание |
|-----------------------|-------------|----------|
| `BOT_TOKEN`           | да          | Токен бота от @BotFather. |
| `WEB_SECRET_KEY`      | да          | Случайная строка для JWT (например: `openssl rand -hex 32`). |
| `BASE_URL`            | да          | Публичный URL приложения **без слэша в конце**. Берётся из Amvera после деплоя: в карточке приложения или в разделе «Домены»/«Ссылка» (например `https://твой-проект.amvera.ru`). Нужен для ссылок в письмах (сброс пароля, приглашения родителей). |
| `INIT_ADMIN_EMAIL`    | нет         | Email первого администратора (при первом старте создаётся, если задано вместе с паролем). |
| `INIT_ADMIN_PASSWORD` | нет         | Пароль первого администратора (не короче 4 символов). |
| `А`| нет         | ФИО (по умолчанию «Администратор»). |

Переменная **`DATABASE_URL` не нужна** — используется SQLite в `/data/school.db`. При желании можно задать свой путь к файлу БД через `SQLITE_DB_PATH` (по умолчанию в `start.sh` подставляется `/data/school.db`).

Секреты (`BOT_TOKEN`, `WEB_SECRET_KEY`) в интерфейсе Amvera по возможности помечай как секреты.

---

## База данных: SQLite в /data

- **Файл БД:** `/data/school.db` (в `start.sh` задаётся `SQLITE_DB_PATH=/data/school.db`). Папка `/data` в Amvera — это постоянное хранилище (`persistenceMount` в `amvera.yml`): при пересборке и перезапуске файлы в ней сохраняются.
- **Таблицы** создаются автоматически при первом запуске приложения (`init_db()`). Настраивать PostgreSQL или создавать БД вручную не нужно.

Бэкап: скопировать файл `/data/school.db` (через панель Amvera или бэкап хранилища). Перенос на другой сервер — см. [BACKUP_MIGRATION.md](BACKUP_MIGRATION.md).

## Проверка перед деплоем

- [ ] Заданы `BOT_TOKEN`, `WEB_SECRET_KEY`, `BASE_URL`.
- [ ] После первого деплоя в `BASE_URL` подставлен реальный URL приложения (без слэша в конце).

---

## Как это устроено

- **start.sh** при старте выставляет `SQLITE_DB_PATH=/data/school.db` (если не задано иначе), при наличии `INIT_ADMIN_*` один раз запускает `scripts/create_administrator.py`, затем в фоне запускает бота (`python app.py &`), затем **главным процессом** — веб (`exec uvicorn ... --port 80`). Amvera направляет HTTP на порт 80.
- Веб и бот используют один и тот же файл SQLite в `/data` — данные общие.

---

## Если что-то не так

- **Веб не открывается:** проверь в логах, что uvicorn слушает порт 80 и что в `amvera.yml` указан `containerPort: 80`.
- **Бот не отвечает:** проверь `BOT_TOKEN` и логи (бот стартует в фоне).
- **Данные пропадают:** убедись, что в `amvera.yml` задан `persistenceMount: /data` и приложение реально пишет в `/data/school.db` (в логах при первом запуске будет строка вида «SQLite (школа) подключена: /data/school.db»).
