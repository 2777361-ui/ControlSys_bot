# Запуск бота на Render.com

Бот запускается как **Background Worker** (постоянно работающий процесс без входящих HTTP-запросов).

## Как этим пользоваться при деплое

После первого деплоя нужно **один раз** создать первого администратора. Дальше войти в веб и через «Пользователи» добавлять остальных.

### Создание администратора

**Способ 1 — переменные окружения (рекомендуется):**  
В Render в разделе **Environment** добавь переменные (можно потом удалить):
- `INIT_ADMIN_EMAIL` — email для входа в веб (например `admin@school.ru`)
- `INIT_ADMIN_PASSWORD` — пароль (не короче 4 символов)
- `INIT_ADMIN_FULL_NAME` — ФИО (например `Иванов Иван`)

Затем один раз открой **Shell** (в карточке сервиса → Shell), выполни:
```bash
python scripts/create_administrator.py
```
Скрипт прочитает данные из переменных и создаст администратора без ввода с клавиатуры.

**Способ 2 — интерактивно:**  
Открой **Shell** в Render и выполни:
```bash
python scripts/create_administrator.py
```
Введи email, пароль и ФИО когда скрипт запросит.

**Способ 3 — при старте воркера (если БД локальная и одна на процесс):**  
Чтобы администратор создался при первом запуске без ручного Shell, задай `INIT_ADMIN_EMAIL`, `INIT_ADMIN_PASSWORD`, `INIT_ADMIN_FULL_NAME` в Environment и в **Start Command** укажи:
```bash
python scripts/create_administrator.py; python app.py
```
Скрипт создаст администратора при каждом рестарте, но в БД останется только один (повторно не добавит). После первого успешного входа в веб переменные можно удалить из Environment.

**Важно:** на Render у Background Worker диск эфемерный — при перезапуске сервиса локальная SQLite может сброситься. **Чтобы данные не пропадали**, используйте внешнюю БД: задайте переменную **`DATABASE_URL`** (см. раздел «База данных: Supabase» ниже). Тогда и бот, и веб работают с одной постоянной БД.

### Дальнейшие шаги

1. Открой веб-интерфейс (если развёрнут отдельно как Web Service) и войди под созданным администратором (email + пароль).
2. В меню: **Пользователи** → **Добавить пользователя**. Создавай нужных пользователей, выбирая роль (Директор, Бухгалтер, Учитель, Столовая, Родитель, при необходимости ещё один Администратор) и задавая пароль.
3. Скрипт `create_director.py` можно использовать вместо администратора, если нужен только первый директор: в Shell выполни `python scripts/create_director.py` и введи данные.

## Шаг 1: Репозиторий на GitHub

Убедись, что проект в Git и запушен на GitHub (или GitLab):

```bash
git add .
git commit -m "Deploy to Render"
git push origin main
```

## Шаг 2: Создать сервис на Render

1. Зайди на [render.com](https://render.com) и войди в аккаунт.
2. **Dashboard** → **New** → **Background Worker**.
3. Подключи репозиторий (GitHub/GitLab) и выбери репозиторий с проектом `Controlsysem_bot`.
4. Настройки:
   - **Name:** например `school-telegram-bot`.
   - **Environment:** **Python 3**.
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `python app.py`
   - **Root Directory:** оставь пустым (корень репозитория).

## База данных: Supabase (данные не пропадают)

По умолчанию используется локальный **SQLite** (`data/school.db`). На Render диск эфемерный — при перезапуске данные могут пропадать. Чтобы этого не было, подключи **PostgreSQL** через Supabase.

### Что нужно сделать (таблицы «активировать» не нужно)

1. Зайди на [supabase.com](https://supabase.com), создай проект (если ещё нет).
2. В проекте: **Project Settings** → **Database** → в блоке **Connection string** выбери **URI** и скопируй строку (типа `postgresql://postgres.[ref]:[PASSWORD]@...supabase.co:5432/postgres`).
3. В Render в **Environment** добавь переменную **`DATABASE_URL`** (отметь как **Secret**) и вставь эту строку, подставив вместо `[YOUR-PASSWORD]` пароль из Supabase (его можно посмотреть/сбросить в том же разделе Database).
4. Задеплой или перезапусти сервис.

**Таблицы создаются автоматически** при первом запуске приложения: код вызывает `init_db()` → при наличии `DATABASE_URL` выполняется создание всех таблиц в Supabase (`CREATE TABLE IF NOT EXISTS`). Вручную ничего создавать в Supabase не нужно.

### После деплоя и редеплоя

- **С заданным `DATABASE_URL`:** приложение подключается к Supabase, данные хранятся там и **не пропадают** при редеплое или перезапуске Render. При первом деплое таблицы создадутся; при следующих деплоях они уже есть — всё работает как обычно.
- **Без `DATABASE_URL`:** используется SQLite на диске воркера; при редеплое диск обнуляется — **данные могут пропасть**.

Если `DATABASE_URL` не задана, по-прежнему используется SQLite (удобно для локальной разработки).

## Шаг 3: Переменные окружения

В разделе **Environment** добавь переменные:

| Переменная       | Описание                    | Секрет |
|------------------|-----------------------------|--------|
| `BOT_TOKEN`      | Токен бота от @BotFather    | Да     |
| `WEB_SECRET_KEY` | Случайная строка для JWT веба (обязательно в проде, не использовать дефолт из кода) | Да (в проде) |
| `DATABASE_URL`   | Строка подключения к PostgreSQL (Supabase) — чтобы данные не пропадали | Да |
| `BASE_URL`       | Публичный URL веб-интерфейса (например `https://your-app.onrender.com`) — для ссылок приглашения родителей и сброса пароля; без слэша в конце | Нет  |

- Нажми **Add Environment Variable**.
- Для `BOT_TOKEN` и `DATABASE_URL` включи **Secret** (значения не будут видны в логах).

## Проверка перед деплоем

- [ ] **BOT_TOKEN** задан в Environment (Secret).
- [ ] **WEB_SECRET_KEY** задан (если поднимаешь веб) — своя случайная строка, не дефолт из кода.
- [ ] **DATABASE_URL** задан (Supabase/PostgreSQL), если нужны постоянные данные.
- [ ] **BASE_URL** задан (URL веб-интерфейса без слэша в конце) — чтобы ссылки приглашения родителей вели на веб, а не в Telegram.
- [ ] Локально: `PYTHONPATH=src python3 -m pytest tests/` — все тесты проходят.

## Шаг 4: Деплой

Нажми **Create Background Worker**. Render соберёт проект и запустит `python app.py`. В логах должны появиться строки о запуске бота и polling.

## Важно

- **База данных:** по умолчанию используется SQLite (`data/school.db`). На Render диск эфемерный — задай **`DATABASE_URL`** (Supabase или другой PostgreSQL), чтобы данные не пропадали (см. раздел «База данных: Supabase» выше).
- **Веб-интерфейс:** если нужно поднять и сайт (FastAPI), создай отдельный **Web Service** с командой запуска `uvicorn web.main:app --host 0.0.0.0 --port $PORT` и теми же переменными окружения (в т.ч. `DATABASE_URL` и `WEB_SECRET_KEY`). Задай **`BASE_URL`** равным URL этого Web Service (например `https://school-web.onrender.com`) — тогда ссылки приглашения родителей будут вести на веб, а не в Telegram.
- **Бесплатный план:** у Background Worker есть ограничения; при долгом простое сервис может засыпать. Для круглосуточной работы бота может понадобиться платный план.

## Если используешь render.yaml

В корне проекта есть `render.yaml`. При создании сервиса можно выбрать **Apply Render Blueprint** и указать этот репозиторий — часть настроек подтянется из файла. Переменную `BOT_TOKEN` всё равно нужно задать вручную в Dashboard (и отметить как Secret).
