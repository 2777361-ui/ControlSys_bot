# Universus — эхобот (Telegram)

Telegram-бот на Python (aiogram), который повторяет всё, что пишет пользователь.

## Запуск

1. Создай бота в [@BotFather](https://t.me/BotFather), получи токен.
2. В корне проекта создай файл `.env` (по образцу `.env.example`):
   ```
   BOT_TOKEN=твой_токен_от_BotFather
   ```
3. Создай виртуальное окружение и установи зависимости:
   ```bash
   cd Bots/Universus_bot
   python3 -m venv .venv
   source .venv/bin/activate   # Windows: .venv\Scripts\activate
   pip install -r requirements.txt
   ```
4. Запуск из корня проекта `Universus_bot`:
   ```bash
   python -m src.bot
   ```
   Либо из папки `src`:
   ```bash
   cd src && python -m bot
   ```

## Переменные окружения

| Переменная            | Описание                    | Пример        |
|-----------------------|-----------------------------|---------------|
| `BOT_TOKEN`           | Токен от BotFather          | (обязательно) |
| `OPENROUTER_API_KEY`  | Ключ OpenRouter для /chat   | (опционально) |

### Деплой на Amvera

Файл `.env` в репозиторий не попадает. Задай переменные в панели Amvera: **приложение → Настройки / Environment → Переменные окружения**. Добавь минимум `BOT_TOKEN`; при использовании `/chat` — также `OPENROUTER_API_KEY`. Запуск: `python app.py` (см. `amvera.yml`).

## Команды бота

- `/start` — приветствие и клавиатура.
- `/help` — справка.
- `/chat` — режим чата с ИИ — предлагает выбрать один из трёх режимов. Выход: `/exit`.
- `/plus1` — прибавить 1 к числу.
- Любой другой текст — бот повторяет его (эхо).

## Режимы чата с ИИ

При нажатии `/chat` появляются три инлайн-кнопки:

| Режим | Что делает |
|-------|-----------|
| 💬 **Ассистент** | Обычный режим — ИИ отвечает как полезный помощник |
| 🎨 **ASCII-арт** | ИИ рисует изображения символами ASCII по запросу |
| 🌐 **Переводчик (RU → EN)** | Пользователь пишет на русском — бот переводит на английский |

## Тесты

Покрыты сервисный слой (текст, числа, конфиг, логирование). Запуск из папки `Universus_bot` с активированным `.venv`:

```bash
pip install -r requirements.txt
python -m pytest tests/ -v
```

Путь `src` подставляется автоматически через `tests/conftest.py`.

## SQLite — история сообщений

Бот хранит историю всех сообщений в базе SQLite (`data/bot_history.db`).
Для каждого сообщения записываются: user_id, username, имя, роль (user/assistant), текст, модель ИИ, сырой ответ и время.

Файл БД создаётся автоматически при первом запуске.

## Суперлогирование

Логи пишутся одновременно:
- **В терминал** (уровень INFO) — основные события.
- **В файл** `logs/bot.log` (уровень DEBUG) — полная детализация.

Что попадает в лог:
- Запуск и остановка бота
- Все входящие события и сообщения от пользователей
- Текущее сообщение пользователя
- Текущая модель ИИ
- Полная история сообщений, отправленных ИИ
- Сырой ответ от ИИ (JSON до обработки)

## Структура

- `src/bot/main.py` — запуск и polling, FSM storage, инициализация БД.
- `src/bot/database.py` — SQLite: хранение истории сообщений по пользователям.
- `src/bot/routers/` — start, help, chat (ИИ), plus1, echo.
- `src/bot/services/` — text, numbers, llm (OpenRouter с fallback по моделям).
- `src/bot/keyboards/` — клавиатуры.
- `src/bot/config.py` — загрузка `.env`.
- `src/bot/utils/logging.py` — суперлогирование (терминал + файл).
- `data/bot_history.db` — SQLite база (создаётся автоматически).
- `logs/bot.log` — лог-файл (создаётся автоматически).
