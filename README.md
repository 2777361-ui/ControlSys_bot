# Школьная система — Telegram-бот и веб-интерфейс

Система для школы (1–11 классы): учёт платежей, столовая, мероприятия, роли (директор, бухгалтер, родитель). Платежи подтверждает только бухгалтер. Есть история платежей и чеков, напоминания о платежах с 1 по 10 число месяца.

## Возможности

- **Роли:** администратор (первый в системе, раздаёт роли и пароли), директор, бухгалтер, родитель, учитель, столовая.
- **Родители:** авторизация по Telegram (если привязан аккаунт) или по email через веб-сайт.
- **Ученики:** 1–11 класс, привязка к родителю.
- **Платежи:** столовая, мероприятия, прочее. Статус — только бухгалтер подтверждает или отклоняет.
- **История:** все платежи и их статусы хранятся в БД.
- **Напоминания:** с 1 по 10 число каждого месяца бот отправляет родителям напоминание о возможности внести средства на питание (баланс столовой).
- **Веб-интерфейс:** вход по email/пароль для директора, бухгалтера, столовой, учителя и родителей. Адаптивный дизайн для ПК и телефона. **«Забыл пароль»** — на почту отправляется ссылка для сброса пароля (нужна настройка SMTP).
- **Пригласительная ссылка для родителей:** в разделе «Ученики» у каждого ученика есть кнопка «Пригласить родителя». Ссылка ведёт на **веб-сайт** (не в Telegram): родитель открывает её в браузере, регистрируется (ФИО, email, пароль) и привязывается к ребёнку. В проде задайте **`BASE_URL`** (URL веб-интерфейса) в переменных окружения, чтобы ссылка была корректной. Ссылка действительна 30 дней.
- **Отделы (только директор):** в разделе «Отделы» директор может создавать отделы и добавлять в них сотрудников (учителей, бухгалтеров и т.д.). Отделы используются для рассылок.
- **Оплата в боте (родители):** кнопка «Оплатить» — выбор ученика, назначение (питание, обучение и т.д.), ввод суммы. Оплата через **Telegram Stars** (комиссия +35% настраивается) и/или через **платёжную систему** (Stripe, YooKassa и др. — токен от @BotFather). Платёж сразу попадает в систему и подтверждается.
- **Рассылки:** директор и бухгалтер могут отправлять сообщения в Telegram по выбору — по учителям и/или по отделам. Учитель может делать рассылку по ученикам (сообщение получают родители выбранных учеников). Сообщения ставятся в очередь и отправляются ботом каждые 30 секунд.

## Запуск

### 1. Окружение и зависимости

```bash
cd Controlsysem_bot
python3 -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

### 2. Переменные окружения

В корне проекта создай `.env`:

```
BOT_TOKEN=токен_от_BotFather
WEB_SECRET_KEY=случайная_строка_для_JWT   # опционально, для веб-сессий

# Оплата в боте (родители): Telegram Stars и/или платёжная система
PAYMENT_PROVIDER_TOKEN=   # токен от @BotFather (Stripe, YooKassa и др.) — если пусто, доступна только оплата Stars
PAYMENT_STARS_COMMISSION_PERCENT=35   # комиссия к сумме при оплате Stars (родитель платит сумма + 35%)
PAYMENT_STARS_RUB_RATE=1.5   # курс: рублей за 1 Star (для перевода суммы в звёзды)
PAYMENT_PROVIDER_CURRENCY=RUB   # валюта для провайдера (RUB, USD и т.д.)

# «Забыл пароль»: отправка ссылки на почту (обязательно для работы восстановления пароля)
SMTP_HOST=smtp.example.com
SMTP_PORT=587
SMTP_USER=your@email.com
SMTP_PASSWORD=пароль_приложения
SMTP_FROM=Школа <noreply@example.com>
BASE_URL=https://your-school.example.com   # адрес сайта для ссылок в письмах (без слэша в конце)
```

### 3. Первый администратор

Сначала создаётся **администратор** — он входит первым и раздаёт роли и пароли остальным (директор, бухгалтер, учитель, столовая, родитель):

```bash
python scripts/create_administrator.py
```

Введи email, пароль и ФИО. Войди на веб-интерфейс по этому логину, открой **«Пользователи» → «Добавить пользователя»** и создай директора, бухгалтера, учителей и т.д., задавая каждому роль и пароль.

При необходимости первого **директора** (без отдельного администратора) можно создать так:

```bash
python scripts/create_director.py
```

#### Восстановление пароля администратора (если забыл пароль и почта недоступна)

Если администратор забыл пароль, а «Забыли пароль» не срабатывает (почта недоступна или SMTP не настроен), пароль можно сбросить **на сервере** скриптом (нужен доступ к окружению проекта и БД):

```bash
cd /path/to/Controlsysem_bot
source .venv/bin/activate
python scripts/reset_admin_password.py
```

Скрипт найдёт учётную запись администратора и запросит новый пароль. Для неинтерактивного запуска (например при деплое) можно задать переменные окружения:
- `ADMIN_RECOVERY_EMAIL` — email администратора, которому сбрасываем пароль (если в системе несколько администраторов)
- `ADMIN_RECOVERY_NEW_PASSWORD` — новый пароль (не короче 4 символов)

Пример: `ADMIN_RECOVERY_EMAIL=admin@school.ru ADMIN_RECOVERY_NEW_PASSWORD=новый_пароль python scripts/reset_admin_password.py`

### 4. Как этим пользоваться при деплое

После деплоя (Render, Amvera или другой хостинг) нужно **один раз** создать первого администратора. Два варианта:

**Вариант А — через переменные окружения (удобно для облака):**  
В настройках сервиса задай переменные (одноразово, потом можно удалить):
- `INIT_ADMIN_EMAIL` — email для входа в веб
- `INIT_ADMIN_PASSWORD` — пароль (не короче 4 символов)
- `INIT_ADMIN_FULL_NAME` — ФИО администратора

Затем один раз выполни на сервере (например через Shell в Render или одну команду в цикле деплоя):
```bash
python scripts/create_administrator.py
```
Скрипт прочитает данные из переменных и создаст администратора без запроса ввода. После этого войди в веб по этому email и паролю.

**Вариант Б — интерактивно:**  
Подключись к серверу по SSH или открой Shell в панели хостинга (например Render → твой сервис → Shell), перейди в каталог проекта и выполни:
```bash
python scripts/create_administrator.py
```
Введи email, пароль и ФИО когда скрипт запросит.

**Дальше (в обоих вариантах):**  
Войди в веб-интерфейс под созданным администратором → открой **Пользователи** → **Добавить пользователя** и создавай нужных пользователей: выбирай роль (Директор, Бухгалтер, Учитель, Столовая, Родитель, при необходимости ещё один Администратор) и задавай пароль. Скрипт `create_director.py` по-прежнему можно использовать, если решите обойтись без отдельного администратора и создать сразу первого директора.

### 5. Запуск бота (локально)

Из **корня проекта** (папка `Controlsysem_bot`):

```bash
# Перейди в папку проекта (подставь свой путь, если другой)
cd /Users/ilapavlov/Documents/Этерна/Bots/Controlsysem_bot

# Активируй виртуальное окружение
source .venv/bin/activate   # Windows: .venv\Scripts\activate

# Установи зависимости, если ещё не ставил
python3 -m pip install -r requirements.txt

# Запуск (используй python3, если команда python не найдена)
python3 app.py
```

Вариант из папки `src`: `cd src && python3 -m bot`.

### 6. Запуск веб-интерфейса

В отдельном терминале **обязательно из папки проекта** (иначе откроется другое приложение, если оно тоже слушает порт):

```bash
cd /Users/ilapavlov/Documents/Этерна/Bots/Controlsysem_bot
source .venv/bin/activate
uvicorn web.main:app --reload --host 0.0.0.0 --port 8000
```

Если порт 8000 уже занят (например, другим проектом), укажи другой порт, например **8001**:

```bash
uvicorn web.main:app --reload --host 0.0.0.0 --port 8001
```

Открой в браузере: **http://localhost:8000** (или http://localhost:8001) — должна открыться страница входа **школьной системы** (поля «Email» и «Пароль»). Логин: email и пароль директора. Если видишь другую админку (например Unibot) — порт занят другим проектом; останови его или используй порт 8001.

## Как проверить работоспособность системы

### Шаг 1: Запуск бота
```bash
cd /Users/ilapavlov/Documents/Этерна/Bots/Controlsysem_bot   # или твой путь к проекту
source .venv/bin/activate

```
В логах должно быть: «Бот: @ваш_бот», «Polling запущен». Если ошибка про `BOT_TOKEN` — проверь `.env`.

### Шаг 2: Первый директор (если ещё не создан)
В **другом** терминале:
```bash
cd Controlsysem_bot && source .venv/bin/activate
python scripts/create_director.py
```
Введи email, пароль, ФИО директора.

### Шаг 3: Запуск веб-интерфейса
В третьем терминале **из папки проекта Controlsysem_bot** (не из Unibot и не из другой папки):
```bash
cd /Users/ilapavlov/Documents/Этерна/Bots/Controlsysem_bot
source .venv/bin/activate
uvicorn web.main:app --reload --host 0.0.0.0 --port 8001
```
Если порт 8000 свободен — можно `--port 8000`. Если там уже Unibot — используй **8001**. Открой **http://localhost:8001** (или 8000) — должна быть форма входа школьной системы, не Unibot Admin. Войди под email и паролем директора.

### Шаг 4: Проверка веба (директор/бухгалтер)
- **Дашборд** — открывается, видны блоки «Платежи, ожидающие подтверждения» и «Родители сообщили: Я совершил платёж».
- **Пользователи** → Добавить пользователя — создай родителя (роль «Родитель», можно без пароля) и запомни или скопируй его Telegram ID (будет после привязки).
- **Ученики** → Добавить ученика — укажи ФИО, класс, выбери только что созданного родителя.
- **Платежи** → Внести платёж — выбери ученика, сумму, назначение (например «Столовая»), сохрани. На дашборде появится платёж «ожидает подтверждения» — нажми «Подтвердить».

### Шаг 5: Проверка бота (родитель)
- В Telegram найди своего бота и отправь **/start**.
- Если твой аккаунт **не** привязан к родителю в системе: бот покажет твой **Telegram ID** — скопируй его.
- В вебе: **Пользователи** → найди созданного родителя → **Изменить / привязать Telegram** → вставь Telegram ID и сохрани.
- Снова в боте отправь **/start** — должно появиться меню: «Мои дети», «Баланс столовой», «История платежей», «Мероприятия», «Я совершил платёж».
- Нажми **Мои дети** — должен отобразиться добавленный ученик и баланс столовой.
- Нажми **Я совершил платёж** — выбери ученика (или сразу отправь). На дашборде в вебе в блоке «Родители сообщили» появится новая строка; проверь кнопки «Внести платёж» / «Платёж уже внесён» / «Проверено, отклонить».

### Шаг 6: Питание и отчёт (по желанию)
- Создай пользователя с ролью **Учитель** или **Столовая** (с паролем), войди под ним на сайт — должен открыться раздел **Питание**. Выбери дату и введи данные по ученикам.
- Под директором открой **Отчёт** — задай фильтры (даты и т.д.) и нажми «Показать»; кнопка «Скачать CSV» должна выдать файл.

Если все шаги проходят без ошибок — система работает корректно.

## База данных

- **Локально:** по умолчанию используется **SQLite** (`data/school.db`) — пользователи, ученики, платежи, мероприятия, напоминания, рассылки. Таблицы создаются при первом запуске.
- **Продакшен (Render и др.):** чтобы данные не пропадали при перезапуске, задай переменную окружения **`DATABASE_URL`** — строку подключения к **PostgreSQL**. Например, **Supabase**: зайди в проект → Project Settings → Database → Connection string (URI), скопируй и подставь пароль. В Render добавь `DATABASE_URL` в Environment (Secret). При первом запуске таблицы создадутся в Supabase автоматически. Подробнее — в [DEPLOY_RENDER.md](DEPLOY_RENDER.md).
- **Бэкап и перенос БД** (например, на Amvera или другой сервер): см. [BACKUP_MIGRATION.md](BACKUP_MIGRATION.md) — как делать дамп PostgreSQL/SQLite, чем открыть и как восстановить на новом хосте.

## Роли и авторизация

| Роль       | Telegram-бот                    | Веб-сайт                          |
|------------|----------------------------------|-----------------------------------|
| Директор   | Подсказка «используйте веб»     | Полный доступ                     |
| Бухгалтер  | То же                            | Дашборд, платежи, ученики, пользователи |
| Родитель   | Меню: дети, баланс столовой, платежи, мероприятия | Вход по email (при желании привязать Telegram) |

**Привязка родителя к Telegram:** родителя добавляют в системе через веб (Пользователи → Добавить). Родитель открывает бота и отправляет `/start` — в ответ бот показывает его **Telegram ID**. Родитель передаёт этот ID в школу; бухгалтер или директор в веб-интерфейсе заходит в «Пользователи» → «Изменить / привязать Telegram» и вносит этот ID. После этого родитель в боте видит меню «Мои дети», «Баланс столовой», «История платежей», «Мероприятия».

## Команды бота (для пользователя)

- `/start` — приветствие; для родителя в базе — меню по детям и платежам; для незнакомого — подсказка и отображение Telegram ID для передачи в школу.
- `/help` — справка по командам. `/plus1` — демо: прибавить 1 к числу.

## Веб-интерфейс (кратко)

- **Дашборд** — список платежей «ожидают подтверждения»; кнопки «Подтвердить» / «Отклонить».
- **Платежи** — вся история платежей.
- **Внести платёж** — выбор ученика, сумма, назначение (столовая/мероприятие/прочее); платёж создаётся в статусе «ожидает», подтверждение — на дашборде.
- **Ученики** — список, добавление нового (ФИО, класс, родитель).
- **Пользователи** — список, добавление (роль, email, пароль), редактирование (привязка Telegram ID).
- **Мероприятия** — список, добавление (название, дата, взнос).

## Напоминания о платежах

Каждый день в **9:00 по Москве** бот проверяет: если сегодня 1–10 число месяца, для каждого ученика отправляет родителю (если у родителя привязан Telegram) одно напоминание в месяц с текстом про баланс столовой. Повторно в том же месяце напоминание не отправляется.

## Оплата в боте (родители)

Родитель нажимает **«Оплатить (Stars / карта)»** → выбирает ученика → назначение (Питание, Обучение и т.д.) → вводит сумму в рублях → способ оплаты:
- **Telegram Stars** — оплата звёздами (в Telegram). К сумме добавляется комиссия (по умолчанию 35%, задаётся в `PAYMENT_STARS_COMMISSION_PERCENT`). Курс рубля к звёздам — `PAYMENT_STARS_RUB_RATE` (по умолчанию 1.5 ₽ за 1 Star).
- **Карта (платёжная система)** — если в конфигурации задан `PAYMENT_PROVIDER_TOKEN` (получить у @BotFather для Stripe, YooKassa и др.). Валюта — `PAYMENT_PROVIDER_CURRENCY` (RUB, USD и т.д.).

После успешной оплаты платёж автоматически создаётся в БД и подтверждается; баланс столовой и отчёты обновляются.

- `src/bot/main.py` — запуск бота, планировщик напоминаний.

- `src/bot/routers/school.py` — обработчики `/start` и меню для родителей (дети, баланс, платежи, мероприятия).
- `src/bot/middleware/auth.py` — подстановка текущего пользователя из БД по telegram_id.
- `src/bot/services/reminders.py` — отправка напоминаний о платежах (1–10 число).
- `web/main.py` — FastAPI: вход, дашборд, платежи, ученики, пользователи, мероприятия.
- `web/templates/` — шаблоны страниц.
- `scripts/create_director.py` — создание первого директора.

## Логирование (аудит и отладка)

- **Файл логов:** `logs/bot.log` (ротация по 5 МБ, до 3 файлов). В консоль выводится уровень INFO, в файл — DEBUG.
- **Формат:** `дата-время | уровень | модуль:строка | сообщение`.
- **Аудит (кто что сделал):** в логах строки с префиксом `AUDIT |`: действие, user_id, role, доп. поля (student_id, amount, path и т.д.). Пароли и токены в лог не попадают (подставляется `***`).
- **Веб:** каждый HTTP-запрос логируется (метод, путь, user_id при наличии, статус ответа, время в мс). Ключевые действия (вход, добавление платежа/ученика, рассылка, подтверждение платежа) пишутся отдельной строкой аудита. При ошибке в обработчике логируется исключение с контекстом (user_id, path).
- **Бот:** в лог попадают вход родителя (/start), регистрация по Telegram, «Я совершил платёж», добавление второго родителя; рассылки — старт задачи, отправка в канал/получателю, успех/ошибка по каждому.
- **БД:** создание и подтверждение платежа, создание рассылки логируются в school_db (payment_id, broadcast_id, created_by).

По логам можно понять, кто что нажал и где произошла ошибка.

## Деплой на Render

Инструкция по запуску бота на [Render.com](https://render.com) как Background Worker — в файле **[DEPLOY_RENDER.md](DEPLOY_RENDER.md)**.

Кратко: New → Background Worker → подключить репозиторий → Build Command: `pip install -r requirements.txt`, Start Command: `python app.py` → в Environment добавить `BOT_TOKEN` (Secret). После деплоя бот будет работать в облаке.

## Деплой на Amvera

В настройках приложения задай переменные `BOT_TOKEN`, при необходимости `WEB_SECRET_KEY`. Запуск бота: `python app.py`. Веб-интерфейс при необходимости разверни отдельно (например, через другой сервис с поддержкой Python/uvicorn).
